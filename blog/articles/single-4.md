# 零阶优化算法浅析

## 引言

在优化问题的世界中，我们通常利用梯度信息来指引搜索方向——从梯度下降法到更复杂的二阶方法（牛顿法），梯度一直是中心角色。然而在某些场景中，**梯度信息不可用或计算困难**。此时，**零阶优化算法**应运而生。本文浅析零阶优化的核心思想和应用。

## 梯度、一阶和零阶优化

### 优化方法的分类

```
按所需信息分类：

零阶方法 (Zeroth-Order)
├─ 只需要目标函数值 f(x)
└─ 不需要梯度信息

一阶方法 (First-Order)
├─ 需要一阶导数 ∇f(x)
└─ 例：梯度下降、Adam 等

二阶方法 (Second-Order)
├─ 需要二阶导数（Hessian 矩阵）
└─ 例：牛顿法、L-BFGS

按复杂度排序：零 < 一 < 二
```

### 为什么需要零阶方法

**梯度不可用的场景：**

```
1. 黑盒优化
   - 只能通过输入得到输出，无法获得内部结构
   - 例：硬件设计优化（每次评估需要物理实验）
   - 例：A/B 测试（只能看到用户反应，不知道数学关系）

2. 非可导情形
   - 目标函数不可导或不连续
   - 例：排序指标（NDCG）、排列问题
   - 例：神经网络离散化后

3. 梯度计算困难
   - 梯度计算代价远高于函数评估
   - 例：某些强化学习问题
   - 例：模型推理时间远小于梯度计算时间

4. 对抗鲁棒性
   - 对抗样本生成中，有时故意不用梯度
   - 例：黑盒对抗攻击（无法获得模型梯度）

5. 硬件与物理约束
   - 设备不支持反向传播
   - 例：某些模拟环境、物理系统
```

**梯度方法的计算成本：**

```
假设模型参数数量 = 10,000

梯度下降：
- 前向传播：1 次
- 反向传播：1 次
- 总计：~2 倍前向传播时间

零阶方法（有限差分）：
- 前向传播：参数数量次（每个参数 ±ε）
- 总计：~20,000 倍前向传播时间

结论：零阶在高维下计算成本很高，需要特殊策略
```

## 零阶优化的核心方法

### 1. 有限差分法 (Finite Differences)

**基本思想**：用差分近似梯度

```
梯度的数学定义：
∂f/∂x_i = lim(ε→0) [f(x + ε*e_i) - f(x)] / ε

实际使用（ε 不取极限）：
∂f/∂x_i ≈ [f(x + ε*e_i) - f(x)] / ε  （单边差分）
∂f/∂x_i ≈ [f(x + ε*e_i) - f(x - ε*e_i)] / (2ε)  （中心差分）

其中 e_i = [0, ..., 0, 1, 0, ..., 0]（第 i 个位置为 1）

Python 示例：
def approximate_gradient(f, x, epsilon=1e-5):
    grad = []
    for i in range(len(x)):
        x_plus = x.copy()
        x_plus[i] += epsilon
        
        x_minus = x.copy()
        x_minus[i] -= epsilon
        
        # 中心差分
        grad_i = (f(x_plus) - f(x_minus)) / (2 * epsilon)
        grad.append(grad_i)
    
    return np.array(grad)

梯度下降更新：
x = x - α * approximate_gradient(f, x)
```

**优缺点：**

```
优点：
- 简单易实现
- 对函数没有特殊要求（只要能计算值）
- 通用性强

缺点：
- 计算复杂度：O(d) 其中 d 是维度
- 数值稳定性问题：ε 太大或太小都会有误差
- 调参困难：ε 的选择很重要
- 在高维空间效率低
```

### 2. 随机搜索 (Random Search)

**基本思想**：随机采样方向，沿着改进方向移动

```
伪代码：
for iteration in range(max_iterations):
    # 在单位球上随机采样方向
    u = 随机向量
    u = u / ||u||  （归一化）
    
    # 评估两个方向上的函数值
    f_plus = f(x + δ*u)
    f_minus = f(x - δ*u)
    
    # 沿着改进方向移动
    if f_plus < f_minus:
        x = x + α*u
    else:
        x = x - α*u

Python 示例：
def random_search_step(f, x, delta=0.1, alpha=0.01):
    # 随机方向
    u = np.random.randn(len(x))
    u = u / np.linalg.norm(u)
    
    # 评估
    f_plus = f(x + delta * u)
    f_minus = f(x - delta * u)
    
    # 更新
    if f_plus < f_minus:
        x = x + alpha * u
    else:
        x = x - alpha * u
    
    return x
```

**收敛性分析：**

```
在凸函数上，随机搜索的收敛速率为 O(1/√T)
其中 T 是迭代次数

对比：梯度下降的收敛速率为 O(1/T) 或更快

结论：随机搜索收敛较慢，但胜在简单通用
```

### 3. 进化策略 (Evolution Strategies)

**基本思想**：模仿自然选择，通过采样和选择优化

```
伪代码：
初始化：种群 P = {x_1, x_2, ..., x_λ}

for generation in range(max_generations):
    for 每个个体 x_i ในP:
        # 变异：加入噪声
        x_i' = x_i + σ * ξ，其中 ξ ~ N(0,I)
        评估 f(x_i')
    
    # 选择：保留最优的 μ 个
    P = 排序后的前 μ 个个体

特点：
- λ (lambda)：每代采样数
- μ (mu)：每代保留数  
- σ (sigma)：变异强度（自适应调整）

Python 示例：
def evolution_strategy(f, x, iterations, lambda_=20, mu=10):
    sigma = 1.0
    
    for gen in range(iterations):
        # 采样
        offspring = []
        for _ in range(lambda_):
            noise = np.random.randn(*x.shape)
            candidate = x + sigma * noise
            offspring.append((f(candidate), candidate))
        
        # 选择
        offspring.sort()  # 按函数值排序
        selected = [x[1] for x in offspring[:mu]]
        x = np.mean(selected, axis=0)  # 新中心
        
        # 自适应 sigma
        sigma *= np.exp(success_rate - 0.2)  # 简化的调整
    
    return x
```

**优点和应用：**

```
优点：
- 并行性好（所有候选可并行评估）
- 鲁棒性强（不易陷入局部最优）
- 无需梯度或光滑性

缺点：
- 收敛速率慢（特别是高维）
- 需要大量函数评估

应用领域：
- 神经网络架构搜索 (NAS)
- 超参数优化
- 对抗鲁棒性提升
```

### 4. 贝叶斯优化 (Bayesian Optimization)

**基本思想**：用概率模型建模目标函数，智能采样

```
流程：
1. 初始化：少量随机评估
2. 拟合模型：用高斯过程 (GP) 或贝叶斯神经网络建模
3. 采集函数：基于模型的不确定性和预测选择下一个评估点
4. 评估：在选中点评估时间哈数
5. 更新模型：加入新数据，重复 2-4

关键是采集函数，常见选项：
- 预期改进 (EI)：选择可能改进最多的点
- 上置信界 (UCB)：平衡利用（已知好点）和探索（不确定点）
- 汤普森采样：从后验采样候选，选择样本的最大值

伪代码（EI）：
for iteration in range(max_iterations):
    # 基于当前观测拟合 GP
    gp.fit(observed_xs, observed_ys)
    
    # 找期望改进最大的点
    x_next = argmax_x EI(x, gp)
    
    # 评估并记录
    y_next = f(x_next)
    observed_xs.append(x_next)
    observed_ys.append(y_next)
```

**特性：**

```
优点：
- 样本高效（少量评估获得好结果）
- 自动权衡探索和利用
- 带有不确定性估计

缺点：
- 模型拟合成本随评估数增加而增加
- 维度过高时效率下降（称为"维度诅咒"）
- 假设函数平滑

最适合：
- 少量预算（评估困难）
- 低维-中等维（<20 维）
- 需要少量次优解的场景
```

## 零阶优化 vs 一阶优化

### 对比总结表

| 方面 | 零阶方法 | 一阶方法 |
|------|--------|--------|
| **梯度需求** | 不需要 | 需要 |
| **计算复杂度** | 高（O(d)个函数评估） | 低（恒定倍数） |
| **收敛速率** | 慢（O(1/√T)） | 快（O(1/T)） |
| **鲁棒性** | 高（不易陷入局部最优） | 中等 |
| **并行性** | 好 | 差 |
| **应用场景** | 黑盒、低维 | 可导、高维 |
| **实现难度** | 简单 | 中等~复杂 |

## 实际案例

### 案例 1：神经网络架构搜索 (NAS)

```
问题：找最好的神经网络架构

为什么用零阶：
- 架构空间很大且离散
- 每个架构评估成本很高（需要训练）
- 梯度信息难以定义

方案：使用进化策略或强化学习
- 表示：架构编码为向量（操作选择、层数等）
- 评估：训练网络获得验证准确率
- 优化：选择性保留好的架构，生成变体
- 结果：找到高效的网络架构

效果：AutoML 使用这些方法自动设计网络，性能媲美人工设计
```

### 案例 2：超参数优化

```
问题：找机器学习模型的最优超参数

为什么用零阶：
- 梯度计算需要反向传播整个超参数空间（不切实际）
- 不同超参数值下的模型差异大

方案：使用贝叶斯优化
- 参数：学习率、regularization、dropout 等
- 目标：最小化验证损失
- 过程：
  1. 随机评估几组超参数
  2. 拟合高斯过程
  3. 选择期望改进最大的组合
  4. 训练模型评估
  5. 重复直到收敛

框架：Bayesopt, Optuna, Ray Tune 等都实现了这套流程
```

## 总结

零阶优化在**梯度不可用**或**计算困难**的场景中发挥关键作用：

1. **方法多样**：从简单的有限差分到复杂的进化策略
2. **权衡取舍**：样本效率 vs 计算复杂度，通用性 vs 专用性
3. **实际应用**：NAS、超参数优化、物理模拟等
4. **发展方向**：
   - 高维问题的零阶方法需突破（通常只适用 <50 维）
   - 与第一阶方法混合（启发式梯度估计）
   - 基于学习的采集函数（用神经网络模型替代 GP）

理解零阶优化有助于解决那些梯度方法难以处理的问题。它提醒我们，虽然梯度很强大，但并非唯一的优化工具。
